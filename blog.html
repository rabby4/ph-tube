Discuss the scope of var, let, and const

"var" is a global scope. let's say if you assign a variable with a var keyword in a block scope or a function. you can access it out of the block scope or function. Also, you can reassign the variable that you assign with the var keyword.

"let" is a block scope. You can't access the "let" value of the block scope. You can re-assign a variable that you assign using "let" keyword. something like "var". But you can't re-declare a variable that you have declared before using "let" keyword.

"const" is something like "let". But the difference is you can't re-assign and re-declare a variable that you have already declared using the "const" keyword

The use cases of "undefined".

  • When we set directly a value "undefined" in a variable.
  • when we declare a variable or set a key in the object but do not assign a value.
  • When we try to access an array index that does not exist in the array.

The use cases of "null".

  • When we set directly a value "null" in a variable.
